Интерфейс управления файловой системой в OS-PI
==============================================

Файловая система -- это абстракция над способом хранения файлов на диске. Она должна включать следующие функции:

* подключение раздела диска;
* получение пути рабочего каталога;
* просмотр содержимого каталога;
* задание рабочего каталога;
* получение информации о файле/каталоге;
* создание/удаление файлов;
* создание/удаление каталогов;
* переименовывание файлов/каталогов;
* открытие/закрытие файлов;
* чтение/запись файлов;
* перемещение указателя чтения/записи;
* информация о свободном месте;
* изменение атрибутов файла.

Система должна сама автоматически определять тип файловой системы при подключении раздела:
::

   (load-partition disk-num part-num)
   ; Загрузить файловую систему из раздела part-num с диска disk-num
   ; Возвращает nil
   ; Возможная ошибка - неправильный номер раздела или ошибка диска

После этой команды дерево файловой системы подключено, и можно работать с файлами.

Пути к файлам в стиле UNIX: /dir/file.txt в виде строки.

Получение пути рабочего каталога:
::

   (cur-dir)
   ; возвращает путь рабочего каталога

Смена рабочего каталога для относительных путей:
::

   (change-dir <путь>)
   ; Возвращает nil
   ; Возможная ошибка - неправильный путь

Просмотр содержимого каталога:
::

   (list-dir <путь>)

Результат выполнения команды просмтотра содержимого каталога содержит список строк для файлов и каталогов:
::

    ("file1.txt"
     "file2.txt"
     "directory")
    ; Возможная ошибка - неправильный путь, ошибка диска

Команда получения информации о файле/каталоге:
::

   (fstat "file.txt")
   ; возвращает хеш-объект
   ((name . "file1.txt")         ; имя файла/каталога
    (size . 424242)              ; размер в байтах
    (create-date . (25 6 1991))  ; дата создания
    (create-time . (25 6 1991))  ; время создания
    (modify-date . (25 6 1991))  ; дата последней модификации
    (modify-time . (25 6 1991))  ; время последней модификации
    (access-date . (25 6 1991))  ; дата последнего доступа
    (access-time . (25 6 1991))  ; время последнего доступа
    (is-dir . t)                 ; флаг каталога
    (другие атрибуты, зависящие от конкретной файловой системы))
   ; Возможная ошибка - неправильный путь, ошибка диска
    
Создание файла:
::

   (create-file <путь>)
   ; создает и открывает файл
   ; возможные ошибки -- некорректный путь,
	; не хватает места, ошибка диска

Удаление файла:
::

   (remove-file <путь>)
   ; удаляет файл
   ; возможные ошибки -- некорректный путь, ошибка диска

Создание каталога:
::

   (create-dir <путь>)
   ; создает каталог
   ; возможные ошибки -- некорректный путь,
	; не хватает места, ошибка диска

Удаление каталога:
::

   (remove-dir <путь>)
   ; удаляет только пустой каталог
   ; возможные ошибки -- некорректный путь, каталог не пуст, ошибка диска

Переименовывание файлов и каталогов:
::

   (rename <путь> <новое имя>)
   ; переименовывает файл/каталог на <новое имя>
   ; возможные ошибки -- некорректный путь, ошибка диска

Проверка на каталог:
::

   (is-directory <путь>)
   ; проверяет является ли объект по пути каталогом
   ; возвращает t, если объект каталог, или nil, если это файл
   ; возможные ошибки -- некорректный путь, ошибка диска

изменение атрибутов:
::

   (set-attr <путь> <новые атрибуты>)
   ; изменяет атрибуты файла на <новые атрибуты>
   ; возможные ошибки -- некорректный путь, ошибка диска, неправильные параметры
		       
Открытие файла:
::

   (open-file <путь>)
   ; возвращает объект-файл для дальнейшей работы
   ; позиция указателя -- в начале файла
	; возможные ошибки -- ошибка пути, ошибка диска

Закрытие файла:
::

   (close-file <file-object>)
   ; возвращает nil
   ; возможная ошибка -- неправильный объект

Чтение файла:
::

   (read-file <file-object> size)
   ; Чтение из файла <file-object> количество байт size
   ; возвращает массив прочитанных байт
   ; возможные ошибки -- ошибка диска, неправильный объект, неправильные параметры

Запись в файл:
::

   (write-file <file-object> buf)
   ; Записать в файл <file-object> массив байт buf
   ; возвращает число записанных байт
   ; возможные ошибки -- ошибка диска, неправильный объект, неправильные параметры

Перемещение указателя чтения/записи в файле:
::

   (seek-file <file-object> offset origin)
   ; Перемещение позиции чтения/записи в файле <file-object> на offset байт начиная от origin
   ; возможные значения origin: SET(начало) CUR(текущее) END(конец)
   ; возможные ошибки -- неправильный объект, неправильные параметры

Получение позиция указателя чтения/записи в файле:
::

   (tell-file <file-object>)
   ; Получение позиции чтения/записи в файле <file-object>
   ; возможная ошибка -- неправильный объект

Сколько осталось свободного места в разделе:
::

   (free-space)
   ; возвращает размер в байтах
   
Реализация интерфейса файловой системы
--------------------------------------

Функции close-file, read-file, write-file, seek-file, tell-file будут работать с файловым объектом - хеш-объект.

Метод close-file достаточно определить пустым, так как файловый объект может быть освобожден сборщиком мусора. В будущих реализациях можно сделать здесь сброс буферов.

Размер блока у разных файловых систем может быть разный. Он определяется при загрузке файловой системы. При этом функции чтения и записи блоков -- одинаковые у всех файловых систем. Поэтому необходима глобальная переменная \*block-sectors\*, которая будет записываться при загрузке и использоваться функциями block-read и block-write. Удобно хранить в этой переменной число секторов по 512 байт для блока. Также нужна глобальная переменная \*block-size\* -- размер блока в байтах, для удобства расчета при чтении или записи. Тогда функции read-file, write-file, seek-file, tell-file могут быть реализованы  одинаково для всех файловых систем, кроме случая обновления атрибутов файла (время модификации, дата доступа). В случае записи в файл может возникнуть увеличение размера файла. В этом случае необходимо добавить новый блок в файл (метод new-block в FileSystem). Размер файла и другие атрибуты при этом также должны измениться. Поэтому нужно сделать класс File с общей функциональностью всех систем, а для конкретной системы делать класс потомок (например Fat32File), где будут действия, специфичные для этой системы.

Метод seek-file просто меняет позицию чтения/записи. Список блоков будет создаваться при открытии файла.

Параметры файла (класс File), которые есть у всех файловых систем:

:name:     имя
:size:     размер файла	   
:position: позиция чтения/записи
:blocks:   список номеров блоков, например (14 15 16)

Остальные параметры добавляются при открытии файла (например, дата/время модификации).

Методы класса File:

* read-file - чтение блоков в буфер, атрибуты меняются в методе потомка;
* write-file - запись блоков из буфера, атрибуты меняются в методе потомка;
* seek-file - меняет указатель чтения/записи, проверяет границы;
* tell-file - возвращает позицию указателя чтения/запси;
* close-file;
  
Для остальных функций нужно сделать класс FileSystem, чтобы они автоматически выбирались в зависимости от загруженной файловой системы. Единственный экземпляр этого класса -- глобальный объект \*file-system\*. Тогда функции интерфейса файловой системы будут вызывать метод глобального объекта:
::

   ; чтобы не было коллизии имен к имени методов добавлять *
   (change-dir "/") -> (change-dir* *file-system* <файловый объект по пути "/">)

Тип файловой системы, а значит и класс будет определяться при загрузке раздела (load-partition).

Методы класса FileSystem делают все операции с объектами файлов.

Сами функции block-read и block-write должны быть реализованы в отдельном модуле. В простой реализации они просто читают и пишут сектора. В более оптимальной реализации они могут использовать буферизацию. Переменная \*block-sector-offset\* задает смещение в секторах места, откуда начинаются блоки. Для чтения секторов есть функции ata-read-sectors и ata-write-sectors.

Работа со структурами внутри блоков
-----------------------------------

Удобно использовать парсеры, которые автоматически связывают имена полей со значениями, используя структуру с именами полей и размером поля в байтах:
::

   (defvar struct '((f1 . 1) ; (имя поля.число байт)
                    (f2 . 4)))

   (setq object (car (funcall (parse-struct struct) (stream-from-arr array nil))))
   ; применяет структуру struct к массиву array и записывает её в переменную object
   (get-hash object 'f1) ; получить значение поля f1

Вспомогательные функции
-----------------------

Разделение строки на список по символу (работа с путями):
::
   
   (split #\/ "/dir1/dir2/file1") -> ("" "dir1" "dir2" "file1")
   (split #\/ "dir2/file1") -> ("dir2" "file1")

Объединение списка с разделителем:
::

   (join #\/ ("dir1" "dir2" "file1")) -> "/dir1/dir2/file1"
   
Класс FileSystem
----------------

Методы:

* fs-init <disk> <start-sector> <sector-count> - загрузка файловой системы
* load-dir <объект-каталог> - раскрытие содержимого каталога
* fstat* <объект-файл/каталог> - получение метаданных файла или каталога
* open-file* <объект-файл> - открытие файла
* new-block <объект-файл/каталог> - выделение нового блока для файла или каталога
* rename* <объект-файл/каталог> <новое имя> - переименовывание файла или каталога
* create-file* <объект-родительский-каталог> <имя файла> - создание нового файла
* remove-file* <объект-родительский-каталог> <объект-файл> - удаление файла
* create-dir* <объект-родительский-каталог> <имя каталога> - создание нового каталога
* remove-dir* <объект-родительский-каталог> <объект-каталог> - удаление пустого каталога
* free-space* - получение свободного места в байтах
* set-attr* <объект-файл/каталог> <новые атрибуты> - изменение атрибутов файла или каталога

Структуру каталогов удобно хранить в естественном виде дерева из файлов и каталогов.
Это будет файловый объект, хранящий необходимую информацию для того, чтобы писать в этот файл или каталог, у которого есть поле dir, которое, если это файл, то равно nil, если это нераскрытый каталог, то равно t, если это загруженный каталог, то это список файловых объектов. Каталоги раскрываются по мере чтения. Функция is-directory возвращает T если файл -- каталог. Глобальный переменная root-directory будет хранить файловый объект корневого каталога с деревом файлов и каталогов. Глобальная переменная working-directory является ссылкой на часть root-directory, хранящая файловый объект текущего каталога. Для раскрытия каталога используется метод виртуальной файловой системы load-dir.
::

   ; Корневой каталог
   Дерево корневого каталога:
    <Файловый объект
    (dir . (
            ("File.txt" . <Файловый объект ; обычный файл
              (dir . nil)>)
            ("UnopenedDirectory" . <Файловый объект ; нераскрытый каталог
              (dir . t)>)
            ("OpenedDirectory" . <Файловый объект ; загруженный каталог
              (dir . (
                      ("File2.txt" . <Файловый объект
                        (dir . nil)>)
                     ))>))
           )>

Работа с путями
---------------

Необходима функция load-path, которая по строке пути возвращает или ошибку (если путь неправильный) или дерево каталога (хеш-объект). По мере раскрытия пути читаются и добавляются в дерево другие каталоги, в файловый объект добавляется раскрытый каталог. Если путь относительный то работа идет с рабочим каталогом. 

При изменении рабочего каталога, искомый путь проверяется и загружается. Загрузка каталога - метод load-dir, на входе список блоков каталога, на выходе объект-каталог. Этот же метод загружает корневой каталог.

функция list-dir использует load-dir для проверки и загрузки пути, а затем отображает дерево каталогов в требуемый список. Каталог это или файловый объект или раскрытый каталог (список).

Запись в файл
-------------

Функция хранит позицию - position и размер записываемого буфера - size. Позиция внутри блока - ofs. Длина записываемого фрагмента - len.

Когда мы пишем внутри блока не переходя границу, при вызове размер становится равным 0 и запись заканчивается:
::

   блок
   ----
   ...
        <- ofs
   ...  len байт
        <- ofs + size
   ----

Когда запись идет с переходом границы блоков, при вызове размер уменьшается по границе, после чего проверяется есть ли у файла следующий блок. Если нет, то для файла выделяется новый блок. Оставшиеся байты будут записаны в следующий блок:
::

   блок
   ----
   ...
        <- ofs
   ...  len  = block-size - ofs
   ---- <- ofs + len

Размер файла увеличивается если position + size > размера файла.
